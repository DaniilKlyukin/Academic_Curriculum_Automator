# Academic Curriculum Automator (ACA)

## Общие сведения
**Academic Curriculum Automator** — это специализированный программный комплекс на базе Python, предназначенный для автоматизации процессов формирования, редактирования и верификации Рабочих Программ Дисциплин (РПД) и иных учебно-методических документов в форматах Microsoft Word и PDF.

Система позволяет минимизировать трудозатраты на выполнение рутинных операций при массовой обработке документации, обеспечивая единообразие оформления и соблюдение локальных нормативных актов образовательной организации.

---

## 🛠 Ключевой функционал

Проект структурирован по функциональным модулям, обеспечивающим полный цикл обработки документа:

### 1. Подготовка и конвертация данных
*   **Трансформация форматов (`.doc` → `.docx`)**: Массовая конвертация устаревших форматов в современные с использованием COM-интерфейсов MS Word.
*   **Экспорт в PDF**: Финализация документов для печати или загрузки в ЭИОС с сохранением всех элементов форматирования.

### 2. Редактирование структурных элементов
*   **Обновление листов согласования**: Автоматизированный поиск и модификация таблиц актуализации. Система идентифицирует нужные секции по ключевым маркерам и производит циклическое добавление периодов (учебных годов).
*   **Коррекция реквизитов и подписей**: Интеллектуальный поиск и замена ФИО ответственных лиц. Алгоритм использует регулярные выражения для корректной обработки инициалов и должностей, ограничивая область действия только зонами подписей.

### 3. Работа с графическими объектами (сканами)
*   **Инъекция графических подписей и печатей**: Автоматическая вставка отсканированных изображений в специфические зоны документа (титульные листы, аннотации, листы согласования).
*   **Контекстный поиск (ScanFinder)**: Система сопоставляет документы и файлы сканов на основе кодов дисциплин, используя настраиваемые шаблоны именования.

### 4. Извлечение и оптимизация
*   **Экстракция аннотаций**: Выделение специфических страниц документа в отдельные PDF-файлы для последующей публикации.
*   **Оптимизация размера (Media Cleanup)**: Удаление избыточных метаданных, дубликатов изображений и сжатие тяжелых медиа-объектов внутри контейнера `.docx`.
*   **Нормализация имен файлов**: Приведение файловой структуры к единому стандарту (удаление спецсимволов, коррекция пробелов).

## 📂 Архитектура проекта

Проект организован по принципу разделения ответственности (Separation of Concerns):

```text
├── core/             # Низкоуровневая логика манипуляции объектами (позиционирование, стили)
├── services/         # Бизнес-логика (обработка подписей, логика листов согласования)
├── run_*.py          # Интерфейсные скрипты (точки входа для конечного пользователя)
└── requirements.txt  # Перечень зависимостей
```

## 💻 Технические требования

Для обеспечения корректной работы комплекса требуются следующие условия:

*   **Операционная система**: Windows 10/11 (взаимодействие с MS Word через библиотеку `pywin32`).
*   **Пакет ПО**: Установленный Microsoft Office (версии 2016 и выше).
*   **Интерпретатор**: Python 3.8 или выше.
*   **Основные зависимости**:
    *   `python-docx` (манипуляция структурой документов)
    *   `pywin32` / `comtypes` (автоматизация Word)
    *   `PyMuPDF` / `pypdf` (обработка PDF)
    *   `fpdf2` (генерация отчетных форм)

## 🚀 Регламент использования

1.  **Развертывание**: Клонируйте репозиторий и установите необходимые зависимости:
    ```bash
    pip install -r requirements.txt
    ```
2.  **Подготовка**: Разместите исходные файлы в рабочей директории. Убедитесь, что файлы имеют формат `.docx`. В случае наличия `.doc`, запустите `run_doc_converter.py`.
3.  **Выполнение**: Запустите необходимый модуль через терминал. Например:
    ```bash
    python run_signature_update.py
    ```
4.  **Валидация**: Следуйте инструкциям в консоли. Введите пути к папкам и параметры обработки.

> **Внимание!** Перед началом массовой обработки рекомендуется создать резервную копию исходных данных. Инструментарий вносит изменения непосредственно в структуру файлов или создает новые копии в зависимости от выбранного режима.

## 📋 Спецификация именования файлов (ScanFinder)

Для корректной работы модуля вставки сканов (`run_scan_insertion.py`) установлен следующий стандарт именования изображений:
`[КОД_ДИСЦИПЛИНЫ]_[ТИП_СТРАНИЦЫ].[РАСШИРЕНИЕ]`

*   `Б1.О.01_1.jpg` — Титульный лист
*   `Б1.О.01_2.jpg` — Лист аннотации
*   `Б1.О.01_3.jpg` — Лист согласования
